# 프로젝트 승격 기준 (Seeds_For_Project)

## 목적
아이디어 풀에서 프로젝트로 분리할 타이밍을 명확히 합니다.

## 최소 요건 (모두 충족)
- 문제 정의가 1문장으로 설명된다
- 사용자/대상이 명확하다
- 기대 성과(측정 가능 KPI)가 1개 이상 있다
- 1주 이내에 할 수 있는 실험/프로토타입 계획이 있다
- 필요한 핵심 부품/기술이 식별됐다

## 권장 요건 (가능하면 충족)
- 비용/리스크 가설이 있다
- 대체 접근 1개 이상이 정리되어 있다
- 실패 기준(Stop 조건)이 있다

## 승격 절차
1. 아이디어를 `Seeds_For_Project`에 이동
2. 요건 체크리스트 작성
3. 통과 시 `35_Projects/프로젝트명/` 생성
4. `00_Requirements`에 초기 요구사항과 KPI 기록

## 비승격 처리
- 요건 미충족 시 `05_Idea_Pool/Clusters`로 되돌림
- 30일 이상 진행 없음: `05_Idea_Pool/Raw`로 이동
