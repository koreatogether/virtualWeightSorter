# 폴더 구조 제안서 ✅

요약: 이 문서는 ‘상상/아이디어 → 정보수집 → 실험 → 구현 → 정리 → 홍보’의 흐름을 따르는, DIY 및 개인 프로젝트에 최적화된 폴더 구조 제안서입니다. 세계 최고 DIYers가 선호하는 관리 방식(작은 프로젝트 단위, 명확한 메타데이터, 재현 가능한 실험 노트, 프로모션 준비 파일 포함)을 반영했습니다.

---

## 핵심 원칙 💡
- **프로젝트 단위로 명확히 분리**: 각 프로젝트는 자체 `README.md`와 메타데이터를 갖습니다.
- **일관된 네이밍 규칙**: 날짜/키워드 기반 파일명(예: `2026-02-06_무게센서_실험.md`) 사용.
- **실험 노트(노트북) 우선**: 매 실험은 재현 가능한 단계로 기록.
- **바이너리 관리**: 큰 바이너리는 Git LFS 사용 권장.
- **프로모션 준비 포함**: 블로그/소셜용 콘텐츠를 함께 버전 관리.

---

## 권장 최상위 폴더 구조 (예) 📁
- `01_IDEAS/` — 상상·아이디어 캡처
  - `inbox.md` (빠른 아이디어 포착)
  - `brainstorm/` (스케치, 컨셉 아카이브)
- `02_RESEARCH/` — 정보수집, 자료, 데이터시트
  - `components/`, `papers/`, `links.md`
- `03_EXPERIMENTS/` — 실험 노트 및 데이터
  - 각 실험은 `YYYY-MM-DD_slug/` 폴더로 구분
  - `notebook.md` (실험 절차, 결과, 사진, 데이터)
- `04_PROTOTYPES/` — 프로토타입 설계·스케치
  - CAD/회로도/부품리스트(BOM)
- `05_PROJECTS/` — 구현된 프로젝트(각 프로젝트는 독립)
  - `project-slug/`
    - `README.md` (한줄 요약, 상태, 목표)
    - `code/`, `hardware/`, `docs/`, `assets/`, `bom.csv`, `release/`
- `06_DOCUMENTATION/` — 가이드·튜토리얼·매뉴얼
- `07_ASSETS/` — 공용 이미지·동영상·템플릿(프로모션용 포함)
- `08_PROMOTION/` — 블로그 초안, 소셜 포스트, 썸네일
- `09_ARCHIVE/` — 종료된 프로젝트, 레거시 자료
- `CONTRIBUTING.md`, `TEMPLATE_PROJECT.md`, `README.md` (루트)

---

## 프로젝트 폴더 내부 템플릿 (예) 🔧
- `README.md` (상태: idea/proto/alpha/beta/released)
- `meta.md` 또는 YAML frontmatter (author, date, tags, license)
- `docs/` (사용자 매뉴얼)
- `code/` (버전별 디렉터리 또는 태그 필요)
- `hardware/` (회로도, PCB 파일)
- `assets/` (고해상도 사진—LFS 권장)
- `experiments/` (실험별 노트와 데이터)
- `promo/` (블로그 초안, 트윗/인스타 문구, 썸네일)

---

## 네이밍 & 날짜 규칙 ✍️
- 실험/노트: `YYYY-MM-DD_설명.md` (정렬과 추적이 쉬움)
- 프로젝트 폴더: `프로젝트-키워드` (영문 소문자 추천: `weight-led-controller`)
- 이미지: `YYYYMMDD_project-slug_01.jpg`

---

## Git & 워크플로 추천 ⚙️
- 브랜치: `main`(안정) / `dev`(통합) / `feature/<slug>`
- 릴리즈 태그 사용 (`v1.0.0`)
- 대용량 파일은 Git LFS에 보관
- 이슈 템플릿: `bug`, `feature`, `experiment`로 분류
- PR 템플릿에 `status: ready/needs-review/wip` 포함

---

## 마이그레이션 간단 가이드 🧭
1. 루트에 위 최상위 폴더들을 생성합니다.
2. 기존 폴더(예: `3dPrint/`, `tutorial/`, `아이디어/`, `Etc/`)를 대응 폴더로 이동하고 각 폴더에 `README.md`를 추가해 현재 상태를 문서화합니다.
3. 중요한 바이너리는 Git LFS로 옮기고 `.gitattributes` 갱신.
4. `TEMPLATE_PROJECT.md`와 `CONTRIBUTING.md`를 만들어 새 프로젝트 시작 템플릿으로 배포.

---

## 권장 관행(짧고 굵게) ✅
- 매 프로젝트는 **짧은 목표**와 **다음 행동 항목**을 README에 둔다.
- 실험은 날짜별로 정리하고, 가능한 한 raw data와 스크립트를 같이 저장.
- 프로모션은 사후 생각이 아닌 배포 준비 프로세스로 포함.
