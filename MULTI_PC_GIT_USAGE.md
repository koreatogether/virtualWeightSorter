# 노트북/데스크탑 병행 작업 Git 사용법

## 작업 시작 전
- 작업 시작 전에 `git fetch` 후 `git pull --ff-only`를 먼저 실행한다.
- 현재 브랜치와 상태를 `git status`로 확인한다.

## 커밋/푸시 규칙
- 가능한 `main` 직접 작업 대신 `feature/...` 브랜치에서 작업한다.
- 커밋 전 `git status`로 변경 파일을 확인하고, 의미 단위로 작게 커밋한다.
- 한 PC에서 push 했으면 다른 PC에서 작업 시작 전에 반드시 pull 한다.

## 충돌/이력 보호
- 같은 파일(특히 문서/로그)을 두 PC에서 동시에 수정하지 않는다.
- 강제 푸시가 필요할 때만 `git push --force-with-lease`를 사용한다.
- `git push --force`는 특별한 경우가 아니면 사용하지 않는다.

## 자동화 주의
- 자동 커밋/자동 push 스크립트 사용 시 PC 간 타이밍 충돌이 생길 수 있다.
- 자동화 사용 시에는 시간대/대상 브랜치 규칙을 고정한다.

## 백업 습관
- 중요한 시점마다 `backup/...` 브랜치 또는 태그를 남긴다.
- 큰 정리 작업 전에는 복구용 브랜치를 먼저 만든다.
