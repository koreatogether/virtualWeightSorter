# ADR-0001: Blynk 플랫폼 제외 및 로컬 웹 서버 기반 모니터링 전환

* **상태**: 승인됨
* **날짜**: 2026-01-21
* **작성자**: GitHub Copilot (사용자 요청 반영)

## 1. 맥락 (Context)
원격 모니터링을 위해 Blynk IoT 플랫폼을 도입하여 연동을 시도했습니다. 하지만 Blynk 앱 상의 데이터 표시 기능이 불안정하고, 플랫폼 자체가 구현하려는 목적에 비해 최적화되어 있지 않다는 판단을 내렸습니다. 이는 하드웨어의 성능 문제가 아니라, 서드파티 앱(Blynk)의 완성도 및 사용성 한계로 인한 결정입니다.

## 2. 결정 (Decision)
* **Blynk 플랫폼 제외**: 복잡하고 불완전한 외부 클라우드 플랫폼(Blynk) 사용을 중단합니다.
* **로컬 웹 서버 구축 (mainRev03)**: Arduino R4 WiFi의 자체 네트워크 기능을 활용하여 로컬 네트워크 내에서 접속 가능한 웹 서버를 직접 구축하기로 결정했습니다.
* **컴퓨터 로컬 웹사이트 모니터링**: 외부 앱 대신 웹 브라우저를 통해 PC나 모바일에서 즉시 데이터를 확인할 수 있는 UI를 직접 설계하고 구현합니다.

## 3. 결과 (Consequences)
* **이점**:
    * 서드파티 서비스 의존성을 제거하여 시스템 독립성 확보.
    * 앱 설치 없이 웹 브라우저만으로 모든 기기에서 모니터링 가능.
    * 원하는 디자인과 차트를 제약 없이 직접 커스터마이징 가능.
* **트레이드오프**:
    * 외부망(LTE/외부 WiFi)에서 접속하려면 포트 포워딩이나 VPN 등 추가적인 네트워크 설정이 필요할 수 있음.
    * 임베디드 웹 서버용 HTML/JS 코드를 직접 작성해야 함.

---
*이 결정으로 인해 기존 `mainRev02`의 Blynk 연동 로직은 유지하되, 차기 버전인 `mainRev03`에서 웹 서버 아키텍처로 완전히 전환합니다.*
