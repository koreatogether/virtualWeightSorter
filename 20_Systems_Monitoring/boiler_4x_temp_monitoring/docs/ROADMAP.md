# Project Roadmap - Boiler Temp Monitoring

본 문서는 보일러 및 분배기 온도 모니터링 시스템의 향후 개발 계획 및 추가될 기능을 정리합니다.

## 1. 사용자 인터페이스 (UI/UX)
- [ ] **물리 버튼을 통한 화면 전환**: 현재의 자동 화면 전환(Rotation) 방식 대신, 또는 병행하여 버튼 입력을 통해 원하는 모드를 선택할 수 있는 기능 추가.
- [ ] **디스플레이 장치 확장**: 현재 20x4 LCD 외에 OLED, TFT LCD, e-paper 등 다양한 시각화 장치 지원.
- [ ] **UI 개선**: 온도 추세를 간단한 그래프나 바 형태로 표시.

## 2. 데이터 처리 및 연결성 (Connectivity)
- [x] **데이터 고정 저장**: PC 수집기(`data_collector.py`)를 통해 최장 7일간의 실시간 데이터를 로컬에 저장하는 기능 구현 완료.
- [x] ~~**Blynk IoT 연동**~~: (플랫폼 사용성 한계로 중단)
- [x] **로컬 웹 서버 구축 (mainRev03)**: Arduino R4 WiFi를 서버로 활용하여 로컬 웹 브라우저에서 실시간 대시보드 구현 완료.
- [ ] **mDNS 도입**: IP 주소 대신 `boiler.local` 같은 이름으로 접속하는 기능 추가.
- [ ] **데이터 연동 확장**: MQTT 등 표준 프로토콜을 통한 로컬 홈서버(Home Assistant 등) 연동 검토.
- [ ] **알림 시스템**: 설정 온도 초과/미만 시 이메일 또는 로컬 알림 전송.

## 3. 센서 및 계산 모듈 확장
- [ ] **추가 센서 통합**: 유량(Flow) 및 압력(Pressure) 센서를 추가하여 실제 사용 열량(kcal) 계산 모듈 구현.
- [ ] **정밀도 향상**: DS18B20 센서의 캘리브레이션 오프셋 설정 기능 추가.

## 4. 고급 분석 (Advanced Analytics)
- [ ] **이상 탐지 (Anomaly Detection)**: 평상시 온도 패턴에서 벗어나는 이상 상황 탐지 알고리즘 구현.
- [ ] **예측 유지보수**: 온도 추세 분석을 통해 열교환기 스케일 생성이나 청소 시점을 제안하는 파이프라인 구축.
