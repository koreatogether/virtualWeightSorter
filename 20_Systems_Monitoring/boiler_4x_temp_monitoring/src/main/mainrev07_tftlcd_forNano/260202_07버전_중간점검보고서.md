# 2026-02-02 다채널 온도 모니터링 시스템 (Ver.07) 중간 점검 보고서

## 1. 상용 장비 대비 필수 기능 평가 (잘 구현된 점)

상용 및 산업용 데이터 로거에서 요구되는 핵심 기능들이 충실하게 구현되어 있습니다.

### ✅ 과거 데이터 리플레이 (Historical Replay) - **[핵심 강점]**
- **평가**: 단순히 현재 온도를 보여주는 것을 넘어, 사용자가 채널을 껐다 켜거나 새로 진입했을 때 **지난 4분간의 데이터**를 즉시 채워 보여주는 기능은 상용 제품에서도 고급 모델에 들어가는 기능입니다.
- **의의**: 데이터 연속성을 보장하여 "끊김 없는 모니터링" 경험을 제공합니다.

### ✅ 완벽한 시점 동기화 (Time-Axis Synchronization) - **[기술적 완성도]**
- **평가**: 채널마다 켜는 시점이 달라도 그래프의 **X축(시간축)**이 흐트러지지 않고 정확히 정렬되도록 `Global Point Count`와 `Lockstep` 로직을 구현했습니다.
- **의의**: 여러 배관의 온도 변화 패턴을 비교 분석할 때 필수적인 기능입니다. 동기화가 안 되면 패턴 분석 자체가 불가능합니다.

### ✅ MCU와 HMI의 효율적 분업 - **[아키텍처 우수성]**
- **평가**: 페이지 이동, 밝기 설정, 절전, 가이드라인 오버레이 등 화면 제어는 Nextion이 전담하고, 아두이노는 오직 데이터 처리에만 집중합니다.
- **의의**: 시스템의 반응 속도(Latency)가 매우 빠르고, 데이터 전송 중 렉이 걸릴 확률을 최소화했습니다.

## 2. 추가하면 좋을 기능 추천 (+Alpha)

현 시스템을 상용 제품 수준으로 끌어올리기 위해 다음 기능 도입을 권장합니다.

### [!] 알람 설정 및 시각적 경고 (Alarm & Indication) - **[강력 추천]**
- **내용**: "상한선 80도 초과 시 해당 채널의 숫자 배경을 빨간색으로 깜빡임" 또는 Nextion의 버저 기능(`beep`) 사용.
- **이유**: 사용자가 화면을 계속 보고 있지 않더라도 문제가 생겼음을 직관적으로 알아야 합니다. (현재 가이드라인은 보기만 할 뿐 경고는 안 함)

### [!] 센서 에러 상태 표시 (Fault Diagnostics)
- **내용**: 센서가 단선되거나 고장 났을 때 `-127` 같은 엉뚱한 값 대신 **"Err"** 또는 **"--"** 표시 및 그래프 그리기 중단.
- **이유**: 잘못된 데이터가 정상 데이터인 것처럼 기록되는 것을 방지해야 합니다.

### 설정값의 영구 저장 (Persistence)
- **내용**: 전원을 껐다 켜도 상/하한선 설정값이나 화면 밝기가 유지되도록 저장. (현재 Nextion의 `Program.s` 초기화 방식은 전원 리셋 시 코드로 적힌 초기값으로 돌아가므로, EEPROM 기능 활용 고려)

## 3. 아쉬운 점 및 개선 권장 사항

### 정밀도와 해상도
- **현황**: 현재 그래프 해상도를 0~255(8비트) 또는 0~100으로 사용 중입니다.
- **제언**: 0.1도 단위의 미세한 변화를 그래프에서 구별하기엔 한계가 있습니다. (다만 HMI 화면 크기상 어쩔 수 없는 부분이기도 합니다.)

### 하드웨어 의존성 명확화
- **현황**: 파일명은 `nano.ino`지만 로직 복잡도는 Mega 2560급 이상의 메모리를 요구하는 방향으로 발전 중입니다.
- **제언**: 만약 Nano 보드에서 계속 운용해야 한다면 SRAM 메모리 부족 현상이 발생할 수 있으므로 주의가 필요합니다.

---

## 4. 총평
현재 시스템은 **취미 수준을 넘어선 준-산업용** 수준의 로직을 갖추고 있습니다. 특히 **동기화와 리플레이 로직**은 현존하는 아두이노-Nextion 예제들 중에서도 최상위권 수준입니다.

센서 연결 후 **"에러 처리"**와 **"알람 기능"**만 추가한다면 실제 보일러실 모니터링 시스템으로 사용하여도 손색이 없을 것으로 판단됩니다.
