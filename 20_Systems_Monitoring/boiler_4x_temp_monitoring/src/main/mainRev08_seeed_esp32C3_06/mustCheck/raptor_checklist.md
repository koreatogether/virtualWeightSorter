# Raptor 체크리스트 (Rev02 → Rev03 준비)

이 문서는 자동 검사 및 인간 입력을 결합하기 위한 체크리스트입니다. 두 파일을 함께 사용하세요:
- `raptor_checklist.json` (기계 판독 가능) ✅
- `raptor_checklist.md` (사람이 읽기 쉬운 체크리스트) ✅

---

## 사용 방법
1. `raptor_checklist.json`의 **null**이나 빈 필드를 테스트 후 채워주세요.
2. 각 항목별로 결과를 적거나 파일 경로(로그, 측정값)를 `attachments`에 추가하세요.
3. 채우신 후 알려주시면 제가 자동으로 결과를 분석하고, 미해결 항목에 대한 구체 작업을 제안합니다.

---

## 항목별 체크리스트

### 1) 전원 (Power) ⚡
- [ ] 5V VBUS에서 측정한 Idle current (mA) 기록
- [ ] 5V VBUS에서 측정한 Peak current (mA) 기록
- [ ] 3.3V 레귤레이터 스펙(최대 출력 mA) 확인
- [ ] 전원 공급원(PC USB, 충전기, 외부 PSU) 명시
- [ ] 부하(Nextion + WiFi)에서 브라운아웃/BOD 발생 여부 확인

### 2) Nextion HMI 💬
- [ ] Nextion 물리 연결 확인 (RX/TX/5V/GND)
- [ ] 레벨 시프터 사용 여부 확인
- [ ] 핸드셰이크(9600→115200) 테스트 통과 여부
- [ ] 복구 시나리오 테스트: 통신 끊김 → 재시도 → 재설정
- [ ] 관련 시리얼 로그 첨부 (파일 경로)

### 3) WiFi / 네트워크 🌐
- [ ] 고정 IP 설정(사용/미사용)
- [ ] WiFi 연결 안정성(24시간 내 끊김 횟수)
- [ ] mDNS가 정상 동작하는지 확인 (boiler.local)
- [ ] NTP 동기화 확인

### 4) 센서 (DS18B20) 🌡️
- [ ] 연결된 센서 개수 기록
- [ ] 각 센서 주소 기록
- [ ] 주기적 읽기값 샘플(타임스탬프 포함)
- [ ] DEVICE_DISCONNECTED_C 발생 시 처리 방식 문서화

### 5) 웹 API / 대시보드 🖥️
- [ ] `/data` 엔드포인트가 정상 동작하는지(응답/타입)
- [ ] `/data` 응답에서 disconnected sensor 처리 여부
- [ ] 대시보드 로그인 비밀번호 환경 설정 가능 여부

### 6) 빌드 / CI 🔧
- [ ] 로컬에서 빌드(PlatformIO) 성공 여부 및 로그 첨부
- [ ] CI(또는 빌드 스크립트) 존재 여부
- [ ] 다른 개발 환경(Windows/Ubuntu)에서 재현성 확인

### 7) 장기/스케일 테스트 🧪
- [ ] HIL(48~72h) Nextion 장시간 테스트 결과
- [ ] 전원 스트레스 테스트 결과
- [ ] WiFi 끊김/복구 테스트 결과
- [ ] data_collector 연동 확인

### 8) 보안/운영 ⚙️
- [ ] 대시보드 비밀번호 외부화(예: `wifiCredentials.h` 대신 보호된 config) 여부
- [ ] 로그(Serial/Web)에서 민감정보가 노출되는지 확인

## 인간 입력(예시 질문 목록)
- 전원: "5V idle = 420 mA, peak = 720 mA (Nextion 3.5" + WiFi Tx)"
- Nextion: "레벨 시프터 사용, 핸드셰이크 실패 로그는 logs/nextion_fail_2026-02-06.log"
- Sensors: "4개 연결, 주소: 28FF..."
