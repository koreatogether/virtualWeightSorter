가장 중요한 setup ()  과 loop () 안에는 중요한 객체던지 각 cpp 에서 선언된 함수들만 존재 할 수 있도록하고 
자체적인 코드는 최대한 절제한다.

각 센서 또는 모듈 의 특징을 잘 나타난 이름과 더불어
이름 + config.h
이름 + 센서면 sensor / 모듈이면 module.h
이름 + 센서면 sensor.cpp / 모듈이면 module.cpp

예를 들어
temp_sensor.h
temp_sensor.cpp
temp_sensor_config.h

이렇게 해서 목록에도 한곳에 모이게 하면 됨

테스트 코드의 원래 목적을 위한 코드인지 ? 
테스트를 통과하기 위한 억지 코드인지 2번 생각해서 
코드를 완성한다.

코딩하다가 모르면 사용자에게 물어본다.
코딩하다가 모르면 구글링한다.
코딩하다가 모르면 레딧 커뮤니티의 관련 글을 찾아서 읽어본다.

코딩 규칙 (초보자 급)

1. 간단한 함수 사용 - 복잡한 클래스 대신 짧고 이해하기 쉬운 함수로 나누기
2. 기본적인 모듈화 - 코드를 논리적으로 그룹화하고, 너무 긴 함수 피하기
3. 절차적 프로그래밍 - setup()과 loop()를 중심으로 순차적으로 생각하기
4. 기본 클린 코드 - 변수명은 의미있게, 주석으로 설명 추가, 들여쓰기 일관되게
5. 아두이노 코딩 가이드 - 핀 번호 명확히, delay() 대신 millis() 사용 권장
6. 함수명은 동사형으로 
7. if문은 2중첩까지만 구현하기
8. switch문 적절하게 사용하기 
9. 라이브러리 불러오기 / 전역 변수 / 셋업 함수 / 루프 함수 / 기타 함수 순서로 하기 
10. 9번 항목에서 코드줄이 250줄이 넘어가면 별도의 파일(.h, .cpp)로 분리하기

코딩 후 규칙
1. 간단한 테스트 - 코드 업로드해서 동작 확인, 에러 없음
2. 가독성 확인 - 다른 사람이 봐도 이해하기 쉽고, 불필요한 코드 제거
