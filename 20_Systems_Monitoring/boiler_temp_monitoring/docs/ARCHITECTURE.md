# Architecture - Boiler Temp Monitoring

## 목적
보일러 및 분배기 온도 데이터를 수집·모니터링하여 이상(과열, 동결, 누수 의심, 과다 사이클링)을 탐지하고, 대시보드 및 알림으로 운영자에게 통보하는 것을 목표로 합니다.

## 구성 요소
- 센서 레이어: 온도 프로브(보일러 출구, 리턴, 탱크, 존별 공급 등)
- 수집기/게이트웨이: 현장 컨트롤러(예: Raspberry Pi / 산업형 PLC)
- 통신: MQTT / HTTP / Modbus(선택)
- 저장/처리: 시계열 DB(InfluxDB 등), 간단 ETL
- 시각화: Grafana 대시보드
- 알림: Prometheus Alertmanager / 커스텀 알림 스크립트

## 데이터 플로우(요약)
1. 센서 → 게이트웨이(수집) → 로컬 캐시
2. 게이트웨이 → 시계열 DB(주기 푸시)
3. DB → Grafana 시각화 및 Alerting

## 확장 포인트
- 유량/압력 센서 추가 시 에너지(열량) 계산 모듈 연동
- 예측 유지보수용 온도 추세 분석 파이프라인 추가
