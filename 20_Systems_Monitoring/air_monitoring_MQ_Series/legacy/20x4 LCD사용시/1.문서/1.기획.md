### 기획 이유
다른 부품 없이 MQ-135 가스센서 모듈을 작동 시킬 수 있는 상황

### 라이브러리 목록
1. 20x4 + Uno R4 WIFI Board - LiquidCrystal_PCF8574.h : https://github.com/mathertel/LiquidCrystal_PCF8574
2. MQ-135 + Uno R4 WIFI Board - MQUnifiedsensor.h : https://github.com/miguel5612/MQSensorsLib
3. DHT22 + Uno R4 WIFI Board - 미정 : 미정
4. SD Card + Uno R4 WIFI Board - SdFat.h : https://github.com/greiman/SdFat

### 준비물
보드 : arduino r4 wifi 호환
센서 : MQ-135 , DHT22
디스플레이 : 20X4 LCD
부저 : 능동부저
기록장치 : SD카드 모듈

### 필요 전압 / 전류 계산
- Arduino r4 wifi 호환 : 5V / 0.1 ~ 0.3A(코드 없는 상태 , 코드 실행시 0.2A, wifi 사용시 0.3A)
- 20X4 LCD : 5V / 0.05A
- DHT22 : 5V / 0.03A
- MQ-135 : 5V / 0.15A
- SD카드 모듈 : 5V / 0.1A
- 부저 : 5V / 0.1A
- 합계 : 5V / 1.53A

###  환경청의 기준 배출 PPM
- 공시 링크 : https://blog.naver.com/PostView.nhn?blogId=dyeng3&logNo=222568123811
- 암모니아(NH3) : 시설마다 틀리지만 엄격한 장소 12 ppm 에서 덜 엄격한 장소는 30 ppm
- 벤젠(C6H6) : 6 ppm 이하
- 알코올(C2H5OH) : 기준 없음

### MQ-135 센서의 감지되는 가스별 PPM 너비값
- 암모니아 : 10 ~ 300 ppm
- 벤젠 : 10 ~ 1000 ppm
- 알코올 : 10 ~ 300 ppm

### AI 추천 받은 소프트웨어 계획
- 기본 로직
    센서 초기화 및 데이터 수집
    모든 센서 데이터를 주기적으로 읽음 (예: 1초 간격)
- 데이터 처리
    센서 데이터(PPM)를 변환 및 평균 처리
    DHT22 온습도 데이터를 함께 기록
- 조건 검출 및 경고
    환경부 기준 초과 시 부저 경고
    실시간 LCD 출력
- 데이터 기록
    SD 카드에 시간, 센서 값, 온습도 값 기록

### 추천 흐름도
[Start]
→ 센서 초기화
→ 센서 데이터 수집
→ 데이터 변환 및 출력
→ 기준 초과 여부 확인
  → 경고 (부저 활성화)
→ 데이터 SD 카드 저장
→ 반복 (루프)
