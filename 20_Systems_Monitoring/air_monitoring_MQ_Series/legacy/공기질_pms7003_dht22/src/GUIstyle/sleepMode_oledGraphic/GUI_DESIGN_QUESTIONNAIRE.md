# PMS7003 OLED GUI 디자인 설문지

Processing으로 구현된 고해상도 GUI를 흑백 OLED(`u8g2`)로 이식하기 위해, 사용 중인 하드웨어 구성과 디자인 선호도를 파악해야 합니다. 아래 항목에 체크(x)해 주세요.

## 1. 하드웨어 사양 (Hardware Specifications)

**Q1. 사용 중인 OLED 디스플레이 모듈은 무엇인가요?**

*   [ ] 1.3인치 SH1106 (128x64)
*   [ ] 0.91인치 SSD1306 (128x32)
*   [ ] 기타 (직접 입력): ________________

**Q2. 조작 방식 (입력 장치)은 어떻게 구성할까요?**
Processing GUI는 마우스 클릭을 사용했습니다. 아두이노에서는 물리 버튼이나 시리얼 명령이 필요합니다.
*   [ ] **시리얼 전용**: 시리얼 모니터로 제어 (현재 `refactored.ino` 방식), OLED는 표시 전용.
*   [ ] **버튼 1개**: 화면/옵션 순환 (Cycle).
*   [ ] **버튼 3개**: 이전, 다음, 선택 (메뉴 조작에 최적).
*   [x] **로터리 인코더**: 돌려서 스크롤, 눌러서 선택.

## 2. 화면 디자인 및 레이아웃 (128x64 픽셀)

**Q4. 화면 구성 방식**
OLED는 작아서 모든 정보(그래프 + 텍스트 + 통계)를 한 번에 보여주기 어렵습니다. 어떻게 구성할까요?
*   [ ] **자동 순환 (Auto-Cycle)**: "상태 뷰(텍스트)"와 "그래프 뷰"를 몇 초마다 자동으로 전환.
*   [x] **수동 전환 (Manual Switch)**: 버튼을 눌러 뷰를 전환.
*   [ ] **단일 화면 (Single Density)**: 필수 숫자와 아주 작은 막대 그래프를 한 화면에 꽉 채워 표시.

**Q5. 막대 그래프 표현 방식**
Processing에서는 3개의 큰 컬러 막대였습니다. 흑백 OLED에서는 어떻게 표현할까요?
*   [ ] **단순 막대**: 값에 따라 높이만 변하는 직사각형.
*   [x] **패턴 채우기**: PM1.0 / 2.5 / 10을 구분하기 위해 빗금 등 패턴 사용.
*   [ ] **텍스트 전용**: 그래픽 없이 "PM2.5: 15 (좋음)" 처럼 큰 글씨로 표시.

**Q6. 통계 표시 (주황/빨강 횟수)**
공기질이 나빴던 횟수(Event Counter)를 OLED에도 표시할까요?
*   [x] 네, 별도의 "통계 화면"을 추가해 주세요.
*   [ ] 아니요, 메모리와 화면 공간 절약을 위해 제거해 주세요.

**Q7. 절전 모드 (Sleep Mode) 화면**
