## 작업 이력 요약

### 1) 초기 페이지 생성
- `index.html`, `style.css`, `app.js` 신규 생성
- 1페이지 구성으로 시뮬레이터 UI 구축
- 컨트롤 버튼 2개 구성:
  - `실행`: 12개 포도 무게 생성/표시
  - `시작`: 조합 탐색 및 결과 시각화

### 2) 시뮬레이터 핵심 기능 구현
- 12개 접시(저울)와 포도송이 그래픽 표시
- 각 포도 중앙에 무게(g) 텍스트 표기
- 무게 범위: `600 ~ 700g` 정수 랜덤
- 조합 조건: `2000g ± 90g`
- 우선순위: 2000g에 가까운 조합 우선
- 다중 박스 표시(가능 시): 최대 4개
- 박스 색상 구분 순서:
  - Box 1: 녹색
  - Box 2: 주황색
  - Box 3: 보라색
  - Box 4: 노랑색
- 결과 정보 표시:
  - 총무게
  - 오차
  - 선택 인덱스

### 3) 반응형/가독성 개선
- 창 크기에 따라 접시 그리드가 유동적으로 재배치되도록 구성
- 결과 리스트가 가로 여유 시 세로 누적 대신 다열 배치되도록 수정
- 접시 선택 시 테두리만 아니라 접시 내부 배경도 박스 색상 계열로 채우도록 변경
- 텍스트 대비(가독성) 개선:
  - 접시 헤더 텍스트 가독성 상향
  - 포도 중앙 무게 텍스트 굵기/그림자 강화

### 4) 난수 엔진 변경
- 사용자 요청으로 난수 생성 로직을 `Math.random()`에서 `XorShift32`로 교체
- 변경 내용:
  - `setSeed(seed)` 추가
  - `xorshift32()` 추가
  - `randInt()`가 XorShift32 출력값을 사용하도록 변경
  - `실행` 버튼 시점에 시간 기반 seed로 초기화
