# HowToOtherDevice: 타 디바이스(노트북 등)에서 분석기 사용 가이드

현재 데스크탑에서 개발/테스트 중인 **Boiler Analyzer**를 노트북이나 다른 컴퓨터에서 가장 쉽고 효율적으로 사용하기 위한 구성 방안에 대한 고민과 가이드입니다.

## 1. 실행 환경의 제약 사항
현재 시스템은 다음 요소들의 조합으로 구성되어 있습니다:
- **백엔드**: Python (`analyzer_server.py`) - 로그 수집 및 API 서버 역할
- **프론트엔드**: HTML/JS (`analyzer_ui.html`) - 차트 및 대시보드
- **로그 저장**: `docs/logs/` 폴더 내 JSON 파일들
- **종속성**: Python 3.x, (선택사항) `requests` 라이브러리

## 2. 노트북에서 사용하는 방법 (시나리오)

### 시나리오 A: 소스 코드 그대로 복사 (가장 유연함)
노트북에 Python이 설치되어 있다면, 프로젝트 폴더 전체(또는 `tools/analyzer_integrated/`와 `docs/logs/`)를 복사하여 바로 사용할 수 있습니다.
- **장점**: 로그 파일이 로컬에 있어 아주 빠름.
- **단점**: Python 설치 및 가상환경 설정이 필요할 수 있음.

### 시나리오 B: 단일 실행 파일(EXE)로 패키징 (가장 편리함)
`PyInstaller` 등을 사용하여 `analyzer_server.py`를 하나의 `.exe` 파일로 빌드할 수 있습니다.
- **방법**: `pyinstaller --onefile --add-data "analyzer_ui.html;." analyzer_server.py`
- **결과**: 사용자는 Python 설치 없이 `.exe` 하나만 실행하면 웹 브라우저가 자동으로 열림.
- **고민**: 로그 데이터 폴더(`./docs/logs`)의 상대 경로 유지가 핵심.

### 시나리오 C: 네트워크 공유 (웹 대시보드 접속)
데스크탑 서버가 켜져 있다면, 노트북에서는 아무것도 설치할 필요 없이 웹 브라우저로만 접속합니다.
- **방법**: 데스크탑 IP가 `192.168.0.100`이라면 노트북 브라우저 주소창에 `http://192.168.0.100:8080` 입력.
- **결과**: 실시간 모니터링 및 데이터 통합 관리에 가장 유리.

## 3. 실행 편의성 개선 아이디어

단순히 폴더를 옮기는 것을 넘어, 다음과 같은 처리를 고려해볼 수 있습니다.

1.  **배치 파일(`.bat`) 활용**: 
    - 사용자는 복잡한 터미널 명령어를 알 필요 없이 `run_analyzer.bat`만 클릭하면 Python 서버 실행부터 브라우저 오픈까지 한 번에 처리되도록 함. (현재 프로젝트에 이미 적용된 방식)
2.  **데이터 휴대성**: 
    - 로그 파일이 많아질 경우를 대비해, 특정 날짜의 로그만 압축해서 옮기거나 외부 클라우드(Dropbox, Google Drive)와 로그 폴더를 동기화하여 어디서든 최신 기록을 보게 함.

## 4. 최종 결론 및 추천
노트북에서 별도의 설정 없이 "파일만 넣어 실행"하고 싶다면, **시나리오 B(EXE 패키징)**와 **시나리오 A(준비된 배포 폴더 + 배치 파일)**의 조합이 가장 현실적입니다.

- **권장 체계**: `[실행파일/배치파일] + [HTML파일] + [로그폴더]` 구성을 하나의 압축파일로 제공.
