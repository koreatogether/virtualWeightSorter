# Final Quality Assurance Test Plan

## 1. 목적
- 공인된 자동화 툴 및 내장 모니터링 도구를 활용하여 코드의 정적 결함, 보안 취약점, 런타임 메모리 안정성을 최종 검증함.

## 2. 검증 항목
### 가. 정적 분석 (Static Analysis)
- **도구**: `pio check` (Cppcheck 기반)
- **내용**: 잠재적 버그(Null 포인터, 버퍼 오버플로우), 미사용 변수, 코딩 표준 위반 사항 검사.

### 나. 메모리 안정성 검사 (Dynamic Analysis)
- **도구**: `MemoryUsageTester` (내장 클래스)
- **내용**: 런타임 중 Heap 메모리 사용량 변화를 모니터링하여 메모리 누수(Memory Leak) 여부 확인.

### 다. 하드웨어 스트레스 및 경계값 테스트
- **방법**: 장시간 구동 테스트 및 임계값(Min/Max) 설정 테스트.
- **내용**: 시스템 가동 시간(Uptime) 유지 및 센서 데이터 처리의 정확성 검증.

## 3. 기록 문서
- `qa_test_report.md`: 각 테스트 단계별 수행 결과 및 조치 사항 기록.
