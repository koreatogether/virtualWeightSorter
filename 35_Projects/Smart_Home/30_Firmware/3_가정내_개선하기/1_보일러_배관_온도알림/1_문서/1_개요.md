# 보일러 배관 온도 모니터링 시스템 개요

## 1. 프로젝트 소개
- 프로젝트 목적
  - 보일러 효율 저하 감지 , 최적 난방 온도 설정의 기초 데이타 수집 
- 기대 효과
  - 매해 온도 차이를 비교함으로써 고장 또는 이상 유무를 미리 감지하여, 한겨울에 비싼 A/S비용을 줄일 수 있음
- 시스템 구성도
  1. 온도 센서에서 온도 감지
  2. 측정된 데이터를 메인보드로 전송
  3. 메인보드에서 Arduino IoT Cloud로 데이터 전송
  4. 사용자는 클라우드에 수집된 데이터를 그래프로 확인

## 2. 하드웨어 구성
- 사용할 보드 및 센서
  - 메인보드: ESP32 x 2 ea
  - 선정사유: 저렴하고 강력한 mcu 성능 , 블루투스 와 wifi 둘다 지원함으로써 변하는 환경에 유연하게 대처 가능 
  - 온도센서: DS18B20 x 2 ea
  - 선정사유 : 값이 싸다 , 단순한 방식 , 방수가능 , 구하기 쉬움 
- 회로 연결 방식
  - 온도센서 DS18B20 1-WIRE 연결 방식 -> 메인보드
- 전원 공급 방식
  - 유선 벽면 콘센트 -> 3.0 usb 아답터 -> 메인보드 
  
- 설치 위치 선정
  - 보일러 출수구 배관  (위치 좀더 자세하게 )
  - 싱크대 밑 배관 ( 환수관 인지 ? 공급관인지 ? )

- led 도입해서 현장 점검 효율화를 꾀할 것인지 ? 

## 3. 소프트웨어 구성
펌웨어 개발 , 어떤 툴을 쓸건지 , 어떤 라이브러리를 쓸건지  , 통신 방식은 어떻게 할건지 등등

여기 부분은

데이타 시각화는 할것인지 ? ( 실시간 , 추세 그래프 )
알림 시스템 할건지 
데이타 저장 및 관리는 어떻게 할것인지 ?
아두이노 클라우드 유료 등급 안내 및 기타 사용 안내 
- thing , property 설정 방법 
- 대시보드 구성방법

마지막으로 데이타 무결성 을 어떻게 입증 할것인지 ??

업데이트는 ota를 꼭 염두에 두고 활용 할것

접근 제어: 이중 인증(MFA) 시스템 도입

센서 끊겼을때 재연결 프로토콜 필수 도입 
데이터 전송 실패시 로컬 저장 후 재전송등을 고려 


## 4. 주요 기능
- 실시간 온도 모니터링
  - 1초 이상의 사용자 정의 주기 가능
- 데이터 로깅
  - Arduino 회원 등급에 따라서 3개월 ~ 1년동안 데이타가 기록되고 다운로드 받아 보관 가능
- 알림 시스템
  - 핸드폰으로 알림 또는 대시보드 가능
- 대시보드
  - 사용자 인터페이스 에 대해서 고민 할것
