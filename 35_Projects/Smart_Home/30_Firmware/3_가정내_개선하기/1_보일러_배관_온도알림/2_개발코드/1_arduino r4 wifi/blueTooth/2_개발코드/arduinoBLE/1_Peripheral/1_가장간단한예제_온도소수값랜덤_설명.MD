# 1_가장간단한예제_온도소수값랜덤_설명

이 문서는 Arduino R4 Wifi 보드에서 실행되는 BLE Peripheral 예제 코드에 대한 설명 문서입니다. 이 예제는 ArduinoBLE 라이브러리를 사용하여 BLE 서비스를 통해 1초마다 -45.0부터 +125.0까지의 온도 값을 소수점 1자리까지 랜덤하게 전송합니다.

## 주요 기능

- **BLE 초기화 및 광고**: 보드가 BLE Peripheral로 동작하도록 초기화하며, 로컬 이름(`ArduinoR4Temp`)으로 BLE 신호를 광고합니다.
- **서비스 및 특성 생성**: 사용자 정의 서비스와 특성을 생성합니다. 특성은 최대 8바이트 크기로, 온도 값을 문자형태로 전송합니다.
- **랜덤 온도 값 생성**: `random(-450, 1251)` 함수를 사용하여 0.1 단위의 랜덤 값을 생성합니다. 예를 들어, -450은 -45.0을, 1250은 125.0을 의미합니다.
- **실시간 데이터 업데이트**: 중앙 기기(예: 스마트폰)가 연결되면 1초마다 온도 값을 업데이트하고 BLE 특성을 통해 전송합니다.

## 코드 구조

1. **헤더 및 UUID 정의**: ArduinoBLE 라이브러리를 포함하고, BLE 서비스와 특성을 위한 UUID를 정의합니다.
2. **BLE 서비스 및 특성 생성**: 사용자 정의 서비스와, 8바이트 크기의 특성을 생성하여 온도 값을 전송합니다.
3. **setup() 함수**: 시리얼 통신과 BLE를 초기화하고, BLE 광고를 시작합니다. 또한, 랜덤 값 생성을 위한 시드값을 설정합니다.
4. **loop() 함수**: 중앙 기기의 연결을 확인하고, 연결되어 있는 동안 매 1초마다 -45.0부터 +125.0까지의 랜덤 온도 값을 생성, 문자형으로 변환한 후 특성을 업데이트합니다.

## 사용 방법

1. 이 코드를 Arduino R4 Wifi 보드에 업로드합니다.
2. BLE 스캐너 앱(예: nRF Connect, LightBlue Explorer)으로 기기를 검색합니다. (광고 이름: "ArduinoR4Temp")
3. 기기에 연결한 후, 특성의 알림을 활성화하면 매 1초마다 업데이트되는 온도 값을 확인할 수 있습니다.

## 참고 사항

- 온도 값은 문자열 형태로 전송되어, 핸드폰에서 직접 숫자로 확인할 수 있습니다.
- 이 예제는 테스트용 랜덤 생성 값을 사용하며, 실제 프로젝트에서는 DS18B20과 같은 온도 센서를 사용하여 데이터를 전송할 수 있습니다.
