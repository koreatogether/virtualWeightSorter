# 품질 및 보안 도구 활용 가이드 (Quality & Security Guide)

이 문서는 프로젝트의 코드 품질 유지와 보안 취약점 방지를 위해 활용 가능한 도구들의 목록과 적용 방법을 설명합니다.

## 1. 정적 분석 도구 (Static Analysis)

### **Cppcheck**
- **용도**: C/C++ 코드의 잠재적 버그(Null Pointer, Buffer Overflow 등) 및 논리 오류 탐지.
- **적용 가능성**: **매우 높음 (이미 사용 중)**
- **사용 방법**:
  - PlatformIO 환경에서 `pio check` 명령어를 실행하면 내부적으로 `cppcheck`를 사용하여 분석을 수행합니다.
  - 현재 프로젝트에서는 `pio check`를 통해 모든 Medium/High 위험군 이슈를 해결한 상태입니다.

### **Codacy (Cloud-based)**
- **용도**: 클라우드 기반 자동 코드 리뷰 도구. GitHub와 연동하여 커밋마다 품질 보고서 생성.
- **적용 가능성**: **높음 (GitHub 업로드 시 추천)**
- **사용 방법**:
  - GitHub에 저장소를 업로드한 후, [Codacy](https://www.codacy.com/) 웹사이트에서 해당 리포지토리를 연결하면 자동으로 작동합니다.
  - 보안, 스타일, 중복 코드 등을 대시보드 형태로 관리할 수 있습니다.

## 2. 보안 스캔 도구 (Security Scanning)

### **TruffleHog**
- **용도**: Git 히스토리나 소스 코드 내에 실수로 포함된 비밀번호, API 키, 토큰 등을 검색.
- **적용 가능성**: **매우 높음 (공개 저장소 푸시 전 필수)**
- **사용 방법**:
  - 로컬 환경에서 `trufflehog [저장소 경로]` 명령어로 실행.
  - 공개 저장소에 올리기 전, 과거 커밋 내역에 비밀번호가 포함되어 있는지 확인하는 데 탁월합니다.

### **Find-Security-Issues (Python 스크립트 등)**
- **용도**: 특정 보안 패턴(위험한 함수 사용 등) 검색.
- **적용 가능성**: **중간**
- **사용 방법**: 이전에 사용하셨던 전용 Python 스크립트가 있다면, 현재 프로젝트의 `src` 폴더를 대상으로 정기적으로 실행하여 보안 결함을 체크할 수 있습니다.

## 3. 코드 지표 분석 (Code Metrics)

### **Code Metrics**
- **용도**: 코드의 복잡도(Cyclomatic Complexity), 유지보수성 지수(Maintainability Index) 측정.
- **적용 가능성**: **높음 (리팩토링 시 유용)**
- **사용 방법**:
  - **VS Code 확장**: "CodeMetrics" 또는 "C++ Metrics" 확장을 설치하여 실시간으로 확인 가능.
  - **Lizard**: Python 기반의 도구로 `lizard src` 명령어를 통해 함수별 라인 수와 복잡도를 분석할 수 있습니다.

## 4. 추천 워크플로우 (Recommended Workflow)

1.  **개발 중**: `pio check`를 수시로 실행하여 런타임 에러 방지.
2.  **커밋 전**: VS Code "Code Metrics"로 함수가 너무 비대해지지 않았는지 체크.
3.  **푸시 전**: `TruffleHog`를 실행하여 민감 정보 포함 여부 확인.
4.  **배포 후**: `Codacy` 대시보드를 통해 지속적인 품질 모니터링.
