# Test Procedure 03: 예외 상황 및 안정성 테스트

이 테스트는 센서 탈락, 입력 지연 등 극한 상황에서의 시스템 복구 능력을 확인합니다.

## 수행 단계

### 1단계: 메뉴 타임아웃
1. 메인 메뉴나 임계값 메뉴에 진입한 상태에서 아무 입력도 하지 않고 60초간 대기합니다.
2. 아래 메시지와 함께 자동으로 `NORMAL_OPERATION` 모드로 복귀하는지 확인합니다:
   ```
   ⏰ Request timeout. Returning to main loop.
   ```

### 2단계: 센서 연결 해제 테스트
1. 시스템이 동작 중인 상태에서 센서 하나를 물리적으로 분리합니다.
2. 약 15초(보고 주기) 후 해당 센서의 상태가 `INVALID` 또는 오류로 표시되는지 확인합니다.
3. 다시 센서를 연결하고 15초 후 정상 온도가 복구되는지 확인합니다.

### 3단계: 잘못된 입력 방어
1. 메뉴 선택 시 숫자 이외의 문자(예: `abc`) 또는 범위를 벗어난 숫자(`9`)를 입력해 봅니다.
2. `Invalid input` 안내 메시지가 나오고 시스템이 예외 종료 없이 대기 상태를 유지하는지 확인합니다.

## 통과 기준 (Pass Criteria)
- 타임아웃 발생 시 무조건 정상 동작 모드로 돌아와야 함.
- 센서 탈락 시 전체 시스템이 멈추지 않고 해당 항목만 오류로 표시되어야 함.
- 잘못된 입력에도 펌웨어가 크래시되거나 루프에 빠지지 않아야 함.
