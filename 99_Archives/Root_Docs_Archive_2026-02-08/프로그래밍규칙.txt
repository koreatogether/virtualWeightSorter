1. 명명 규칙

변수, 메서드, 클래스 등 모든 요소에 의미 있고 설명적인 이름 사용.

클래스와 메서드는 PascalCase, 변수와 매개변수는 camelCase 사용.

일관된 명명 규칙 유지.

2. 코드 구조와 가독성

단순하고 간결한 코드 유지, 작은 단일 목적 기능 지향.

매직 넘버 대신 상수나 설정 값 사용.

잘 명명된 메서드·클래스로 논리 분리.

주석은 목적과 사용법을 설명하며, 불필요한 주석은 지양.

불필요한 코드 제거로 깨끗한 코드 유지.

적절한 들여쓰기와 서식으로 가독성 향상.

코드 블록과 논리적 구분 사이에 빈 줄 추가.

3. 객체 지향 원칙 (SOLID)

단일 책임 원칙: 메서드/클래스는 하나의 책임에 집중.

개방-폐쇄 원칙: 변경에 닫히고 확장에 열려있도록.

리스코프 치환 원칙: 상위 타입을 하위 타입으로 대체 가능해야 함.

인터페이스 분리 원칙: 클라이언트가 사용하지 않는 메서드 의존 방지.

의존 역전 원칙: 구체 클래스보다 추상에 의존.

4. 품질 향상 규칙

깊은 중첩·복잡한 구조 피하기.

불변 상태를 선호하고 지역 변수를 최소화.

단위 테스트 작성, 경로·엣지 케이스 모두 테스트.

DRY(중복 금지) 원칙 준수.

명확하고 의미 있는 매개변수 이름 사용.

예외는 의미 있는 메시지와 함께 처리.

디자인 패턴 적절히 적용.

지속적으로 코드 리팩터링.
