## hd44780 및 hd44780_I2Cexp 라이브러리 상세 분석

Arduino 환경에서 I2C 인터페이스를 사용하는 캐릭터 LCD를 제어할 때 널리 사용되는 `hd44780` 라이브러리와 `hd44780_I2Cexp` I/O 클래스의 GitHub 저장소 주소와 전체 메서드 목록을 제공합니다.

### GitHub 저장소 주소

해당 라이브러리의 공식 GitHub 저장소는 다음과 같습니다.

* **[https://github.com/duinoWitchery/hd44780](https://github.com/duinoWitchery/hd44780)**

이 저장소는 Bill Perry (bperrybap)가 개발했으며, 기존의 LiquidCrystal 라이브러리를 확장하여 더 많은 기능과 하드웨어 인터페이스를 지원하는 것이 특징입니다. `hd44780_I2Cexp`는 이 라이브러리에 포함된 I/O 클래스 중 하나로, PCF8574와 같은 I/O 확장 칩을 통해 LCD를 제어하는 역할을 합니다.

---

### 메서드 목록 (Methods List)

라이브러리는 크게 두 부분으로 나눌 수 있습니다. `hd44780` 기본 클래스는 LCD의 핵심 기능을 제어하며, `hd44780_I2Cexp` 클래스는 I2C 통신과 관련된 부분을 담당합니다.

#### **1. `hd44780` (기본 클래스) 메서드**

이 메서드들은 실제 LCD 화면에 내용을 표시하고 제어하는 핵심적인 기능들을 포함합니다.

##### 핵심 메서드

| 메서드                | 설명                                                                                |
| --------------------- | ----------------------------------------------------------------------------------- |
| `begin(cols, rows)`   | LCD의 크기(열, 행)를 지정하고 초기화합니다.                                         |
| `print(data)`         | 지정된 데이터를 LCD에 출력합니다. 다양한 데이터 타입(문자열, 숫자 등)을 지원합니다. |
| `setCursor(col, row)` | 커서의 위치를 지정된 열(column)과 행(row)으로 이동시킵니다.                         |
| `clear()`             | LCD 화면의 모든 내용을 지우고 커서를 (0, 0) 위치로 이동시킵니다.                    |
| `home()`              | 커서를 (0, 0) 위치로 이동시킵니다. 화면의 내용은 지워지지 않습니다.                 |
| `write(data)`         | 단일 문자(byte)를 LCD에 씁니다.                                                     |

##### 전체 메서드 목록

**초기화 및 상태**
* `begin(cols, rows)`: LCD를 초기화합니다. 성공 시 0을 반환합니다.
* `init()`: 하위 레벨의 초기화를 수행합니다. `begin()`에 의해 내부적으로 호출됩니다.
* `status()`: 현재 LCD의 상태를 반환합니다.
* `fatalError(message)`: 심각한 오류가 발생했을 때 메시지를 출력하고 프로그램을 멈춥니다.

**화면 및 커서 제어**
* `clear()`: 화면을 지웁니다.
* `home()`: 커서를 홈 위치(0, 0)로 이동합니다.
* `setCursor(col, row)`: 커서 위치를 설정합니다.
* `cursor()`: 커서를 보이게 합니다.
* `noCursor()`: 커서를 숨깁니다.
* `blink()`: 커서를 깜빡이게 합니다.
* `noBlink()`: 커서 깜빡임을 멈춥니다.
* `display()`: 화면 표시를 켭니다.
* `noDisplay()`: 화면 표시를 끕니다.
* `scrollDisplayLeft()`: 화면 내용을 왼쪽으로 한 칸 스크롤합니다.
* `scrollDisplayRight()`: 화면 내용을 오른쪽으로 한 칸 스크롤합니다.
* `autoscroll()`: 텍스트가 추가될 때마다 화면이 자동으로 스크롤되도록 설정합니다.
* `noAutoscroll()`: 자동 스크롤 기능을 끕니다.
* `leftToRight()`: 텍스트가 왼쪽에서 오른쪽으로 써지도록 설정합니다. (기본값)
* `rightToLeft()`: 텍스트가 오른쪽에서 왼쪽으로 써지도록 설정합니다.

**데이터 출력**
* `print(data)`: 다양한 타입의 데이터를 문자열로 변환하여 출력합니다.
* `write(data)`: 바이트 단위의 데이터를 직접 씁니다.
* `createChar(num, data)`: 사용자 정의 문자(CGRAM)를 생성합니다. `num`은 0-7 사이의 문자 번호, `data`는 8바이트의 문자 데이터 배열입니다.

**고급 기능**
* `setBacklight(value)`: 백라이트의 밝기를 조절합니다. (I/O 클래스에서 지원해야 함)
* `getBacklight()`: 현재 백라이트의 상태를 가져옵니다. (I/O 클래스에서 지원해야 함)
* `read()`: LCD의 RAM에서 데이터를 읽어옵니다. (일부 하드웨어에서만 지원)
* `setCommandTiming(...)`: 명령어 실행 타이밍을 직접 설정할 수 있습니다.
* `setPowerTiming(...)`: 전원 인가 관련 타이밍을 설정할 수 있습니다.

#### **2. `hd44780_I2Cexp` (I/O 클래스) 메서드**

이 클래스는 `hd44780` 객체를 생성할 때 사용되며, I2C 통신에 특화된 기능을 가집니다.

##### 핵심 메서드 (생성자)

| 메서드                                             | 설명                                                                                  |
| -------------------------------------------------- | ------------------------------------------------------------------------------------- |
| `hd44780_I2Cexp(i2c_addr, i2c_bus, expander_type)` | I2C LCD 객체를 생성합니다. I2C 주소, I2C 버스, I/O 확장 칩 종류를 지정할 수 있습니다. |
| `hd44780_I2Cexp()`                                 | I2C 주소를 자동으로 탐색하여 객체를 생성합니다. 가장 일반적인 사용법입니다.           |

##### 전체 메서드 목록

주로 객체 생성과 관련된 생성자(Constructor)와 내부적인 I/O 처리를 위한 메서드들로 구성됩니다. 사용자가 직접 호출하는 경우는 드물며, 대부분 `hd44780`의 기본 메서드를 통해 간접적으로 사용됩니다.

* `hd44780_I2Cexp(...)`: 다양한 매개변수를 받는 여러 버전의 생성자가 있습니다. I2C 주소, `Wire` 객체, 핀 매핑 등을 지정할 수 있습니다.
* `ioinit()`: I/O 인터페이스를 초기화합니다. `begin()`에 의해 내부적으로 호출됩니다.
* `iowrite(value, type)`: LCD로 데이터를 전송합니다.
* `ioread(type)`: LCD로부터 데이터를 읽어옵니다.
* `iosetBacklight(value)`: I/O 확장 칩을 통해 백라이트를 제어합니다. `setBacklight()`에 의해 호출됩니다.
* `iogetBacklight()`: 현재 백라이트 제어 상태를 가져옵니다. `getBacklight()`에 의해 호출됩니다.

실제 사용 시에는 `hd44780_I2Cexp lcd;`와 같이 객체를 선언한 후, `lcd.begin(16, 2);`, `lcd.print("Hello");` 와 같이 `hd44780`의 기본 메서드를 사용하여 LCD를 제어하게 됩니다.
