# 🚀 시나리오 4~8 구현 가이드 - 먼저 읽어주세요!

## 📋 개요
DS18B20 온도 센서 테스트 프레임워크의 시나리오 4~8 구현을 위한 종합 가이드입니다.  
시나리오 1, 2, 3 구현 과정에서 발견된 공통 패턴과 문제 해결 방법을 정리하여 효율적인 구현을 돕습니다.

## ✅ 현재 상태
### 완료된 시나리오:
- ✅ **시나리오 1**: 센서 테이블 표시 기능
- ✅ **시나리오 2**: 메뉴 진입 및 취소 기능  
- ✅ **시나리오 3**: 개별 센서 ID 변경 기능

### 구현 대상 시나리오:
- 📝 **시나리오 4**: 여러 센서 ID 선택적 변경 
- 📝 **시나리오 5**: 센서 ID 자동 할당
- 📝 **시나리오 6**: 센서 ID 전체 초기화 및 재설정
- 📝 **시나리오 7**: 잘못된 명령어/입력 처리
- 📝 **시나리오 8**: 센서 미연결/오류 상황 처리

## 📁 생성된 가이드 문서들

### 📋 `quick_checklist.md` - 빠른 구현 체크리스트
**용도**: 새 시나리오를 구현할 때 단계별로 체크할 수 있는 실용적 가이드
**포함 내용**:
- ✅ 단계별 구현 체크리스트 (테스트 파일 생성 → test_main.cpp 수정 → run_test.bat 수정 → 테스트 실행)
- 🐛 문제 해결 빠른 참조 가이드
- 📊 완료된 시나리오 추적 목록
- 🎯 한 번에 여러 시나리오 구현 시 팁

**사용 시기**: 🔥 **시나리오 구현할 때마다 반드시 참조!**

### 📖 `scenario_implementation_guide.md` - 종합 구현 가이드  
**용도**: 시나리오 1, 2, 3에서 발견된 공통 패턴과 해결책을 상세히 설명
**포함 내용**:
- 🔄 공통 구현 패턴 (테스트 파일 구조, test_main.cpp 수정 패턴, run_test.bat 수정 패턴)
- ⚠️ 주요 문제점과 해결책 (링킹 에러, 결과 파일 중복, 컴파일 순서)
- 🚀 시나리오 4-8 구현 체크리스트
- 📝 실행 명령어 및 결과 파일 구조

**사용 시기**: 패턴을 이해하고 싶을 때, 문제가 발생했을 때

### 🎯 `scenario_summary_and_priority.md` - 시나리오 요약 및 우선순위
**용도**: 시나리오 4~8의 내용과 구현 순서를 결정하는 가이드
**포함 내용**:
- 📋 시나리오 4~8 각각의 목적, 절차, 복잡도 요약
- 🚀 권장 구현 순서 (쉬운 것부터 어려운 것 순)
- 📅 단계별 구현 계획 및 예상 소요 시간
- 🔄 반복 패턴 활용 방법

**사용 시기**: 구현 순서를 결정할 때, 전체 계획을 세울 때

### 🔧 `scenario_template.md` - 구현 템플릿
**용도**: 복사해서 바로 사용할 수 있는 코드 템플릿 모음
**포함 내용**:
- 📄 `test_scenarioX.cpp` 완전한 템플릿

- ⚙️ `run_test.bat` 추가 코드 템플릿 (컴파일, 링킹)

**사용 시기**: 실제 코드를 작성할 때

## 🚀 권장 구현 순서

### 1순위: 📝 시나리오 5 (센서 ID 자동 할당) - 가장 쉬움 ⭐⭐
**이유**: 가장 단순한 로직, 기존 패턴과 유사, 높은 성공 확률  
**예상 소요 시간**: 1-2시간

### 2순위: 📝 시나리오 6 (센서 ID 전체 초기화) - 쉬움 ⭐⭐  
**이유**: 초기화 로직은 비교적 단순, 기존 코드 재사용 가능  
**예상 소요 시간**: 1-2시간

### 3순위: 📝 시나리오 4 (여러 센서 ID 선택적 변경) - 중간 ⭐⭐⭐
**이유**: 시나리오 3과 유사하지만 여러 센서 처리, 입력 파싱 로직 추가 필요  
**예상 소요 시간**: 2-3시간

### 4순위: 📝 시나리오 7 (잘못된 명령어/입력 처리) - 어려움 ⭐⭐⭐⭐
**이유**: 다양한 에러 케이스 처리 필요, 복잡한 검증 로직 필요  
**예상 소요 시간**: 3-4시간

### 5순위: 📝 시나리오 8 (센서 미연결/오류 상황 처리) - 가장 어려움 ⭐⭐⭐⭐⭐
**이유**: 하드웨어 의존적 테스트, Mock 객체 복잡한 설정 필요  
**예상 소요 시간**: 4-6시간
