# 프로젝트 DS18B20 펌웨어 - 마스터 테스트 체크리스트

**문서 버전:** 1.0  
**기준 펌웨어 버전:** v6.08+  
**작성일:** 2025-07-26

## 1. 개요
이 문서는 DS18B20 센서 네트워크 관리 펌웨어의 모든 기능이 안정적으로 동작하는지 검증하기 위한 종합 테스트 절차를 정의합니다. 각 테스트 항목은 기능별로 분류되어 있으며, 예상 결과와 함께 성공 여부를 기록할 수 있도록 구성되었습니다.

---

## 2. 테스트 사전 준비
- [ ] **하드웨어:** Arduino UNO R4 WiFi 보드에 1개 이상의 DS18B20 센서를 연결합니다. (다양한 시나리오를 위해 3개 이상 권장)
- [ ] **소프트웨어:** 최신 버전의 펌웨어를 보드에 업로드합니다.
- [ ] **모니터링:** Arduino IDE의 시리얼 모니터를 9600bps로 설정하여 연결합니다.

---

## 3. 테스트 시나리오 및 체크리스트

### 3.1. 핵심 UI 및 기본 기능

| **UI-2** | **주기적 상태 보고** | 약 15초 간격으로 센서 상태 테이블이 자동으로 갱신되어 출력된다. | [ ] | |

| **UI-4** | **메뉴 안정성 (반복 테스트)** | `menu` 진입 후 `5`번(취소)을 5회 이상 반복해도, 지연이나 입력 먹통 현상 없이 즉각적으로 반응한다. | [ ] | |
| **UI-5** | **메뉴 타임아웃 (⭐핵심)** | 메뉴 화면에서 60초 동안 아무 입력이 없으면 "Menu timeout." 메시지와 함께 자동으로 일반 모드로 복귀하고, 센서 테이블이 즉시 출력된다. | [ ] | |

### 3.2. ID 관리 기능

#### 3.2.1. 개별 ID 변경 (메뉴 1)
| 테스트 항목 ID | 테스트 내용 | 예상 결과 | 상태 | 비고 |
|---|---|---|:---:|---|
| **IND-1** | **정상 ID 변경** | ID가 할당된 센서를 선택 시 "(Current ID: X)"가 정확히 표시되고, 유효한 새 ID 입력 시 성공적으로 변경된다. | [ ] | |
| **IND-2** | **ID 중복 할당 시도** | 이미 사용 중인 ID를 입력하면 "ID already in use." 메시지가 출력되고 재입력을 요구한다. | [ ] | |
| **IND-3** | **ID 미할당 센서 부재 시** | ID가 할당된 센서가 하나도 없을 때 메뉴에 진입하면, "No sensors with valid IDs found." 메시지를 출력하고 주 메뉴로 복귀한다. | [ ] | |

#### 3.2.2. 선택적 ID 변경 (메뉴 2)
| 테스트 항목 ID | 테스트 내용 | 예상 결과 | 상태 | 비고 |
|---|---|---|:---:|---|
| **SEL-1** | **입력 파싱 (⭐핵심)** | `1 2`, `1,2`, `12` 등 다양한 형식의 숫자 입력이 "Selected sensors: 1, 2" 와 같이 정확히 파싱된다. | [ ] | 두 자리 수, 붙여쓰기 포함 |
| **SEL-2** | **순차적 ID 변경** | 선택된 여러 센서에 대해 순차적으로 새 ID를 문제없이 할당하고, 최종적으로 "All selected sensors have been updated." 메시지를 출력한다. | [ ] | |
| **SEL-3** | **ID 재변경** | 이미 ID가 변경된 센서들을 다시 선택하여 다른 ID로 재변경하는 과정이 정상적으로 수행된다. | [ ] | |
| **SEL-4** | **메뉴 즉시 취소 (⭐핵심)** | 센서 선택 화면에서 `0` 입력 시, 지연 없이 즉시 "Cancelling..." 메시지와 함께 주 메뉴로 복귀한다. | [ ] | |

#### 3.2.3. 주소 기준 자동 ID 부여 (메뉴 3)
| 테스트 항목 ID | 테스트 내용 | 예상 결과 | 상태 | 비고 |
|---|---|---|:---:|---|
| **AUTO-1**| **자동 ID 할당** | 센서의 고유 주소(Address) 오름차순으로 정렬하여 ID를 1번부터 순서대로 부여하고, "Auto-assignment complete." 메시지와 함께 결과를 출력한다. | [ ] | |

#### 3.2.4. 전체 ID 초기화 (메뉴 4)
| 테스트 항목 ID | 테스트 내용 | 예상 결과 | 상태 | 비고 |
|---|---|---|:---:|---|
| **RST-1** | **ID 초기화** | 모든 센서의 ID를 0으로 초기화하고, "--- Connected Sensors (Address List) ---" 테이블을 통해 주소 목록을 보여준다. | [ ] | |
| **RST-2** | **초기화 후 메뉴 진입** | ID 초기화 후 1번 또는 2번 메뉴 진입 시도 시, ID 할당이 필요하다는 안내 메시지를 출력하고 주 메뉴로 복귀한다. | [ ] | |

### 3.3. 예외 상황 처리
| 테스트 항목 ID | 테스트 내용 | 예상 결과 | 상태 | 비고 |
|---|---|---|:---:|---|
| **EXC-1** | **센서 미연결 부팅** | 센서가 하나도 연결되지 않은 상태에서 부팅 시, "No DS18B20 sensors found." 메시지를 출력하며 무한 대기 상태에 진입한다. | [ ] | |
| **EXC-2** | **잘못된 입력 처리** | 메뉴 및 각 단계에서 범위를 벗어난 숫자나 문자를 입력 시, "Invalid input." 관련 메시지를 출력하고 재입력을 요구한다. | [ ] | |
| **EXC-3** | **동적 연결/해제** | 시스템 동작 중에 센서를 연결하거나 제거해도, 다음 주기적 업데이트 시 시스템이 멈추지 않고 상태를 정상적으로 반영한다. | [ ] | |
