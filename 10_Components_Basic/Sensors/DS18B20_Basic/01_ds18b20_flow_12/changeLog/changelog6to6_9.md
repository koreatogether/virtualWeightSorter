## GitHub Actions 기반 CI/CD 목표 달성을 위한 상세 단계 계획

1. 저장소에 `.github/workflows` 폴더 생성
2. 기본 워크플로우 파일(`ci.yml` 등) 생성 및 커밋
	- 예시: 빌드, 테스트, 코드 커버리지, XML 출력 등 단계 포함
3. 워크플로우에서 사용하는 빌드/테스트 스크립트 준비
	- 예: Arduino 빌드 명령, 유닛 테스트 실행, 결과 파일(XML 등) 생성
4. 워크플로우 파일에 각 단계(Job/Step) 명확히 정의
	- OS 매트릭스(Linux, macOS 등) 설정
	- 의존성 설치(예: 라이브러리, 패키지)
	- 빌드 및 테스트 실행
	- 테스트 결과 및 커버리지 리포트 업로드
5. XML 출력 결과를 아티팩트로 저장하거나 PR 코멘트에 첨부
6. 커밋/PR 발생 시 자동 실행되는지 확인
7. 실패 시 알림(이메일, PR 코멘트 등) 설정
8. 필요시 추가 워크플로우(배포, 릴리즈 등) 확장

- 워크플로우 예시 및 템플릿은 GitHub Actions 공식 문서 참고
- 프로젝트 특성에 맞게 단계 및 스크립트 수정
