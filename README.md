# Weigher Sorter Virtual Simulator

`virtualWeightSorter`는 포도 무게를 기준으로 박스 조합을 찾는 가상 분류 시뮬레이터입니다. 12개의 접시에 무작위 포도 무게를 생성한 뒤, 목표 중량과 허용 오차에 맞는 조합을 탐색해 결과를 보여줍니다.

## 핵심 기능

- 12개 접시의 포도 무게를 무작위로 생성
- 목표 중량, 허용 오차, 박스당 개수 조건 설정
- 조건에 맞는 조합을 최대 4개까지 탐색
- 선택된 조합을 접시 색상과 결과 목록으로 표시
- 추천색 문서 기준의 다크 테마 UI 적용

## 파일 구조

- `index.html` : 메인 화면
- `style.css` : 화면 스타일
- `app.js` : 무게 생성, 조합 탐색, 화면 렌더링 로직
- `추천색.md` : 색상/톤 참고 문서
- `DEPLOY.md` : GitHub Pages 배포 참고 문서
- `v002/` : 이전 버전 보관 폴더

## 실행 방법

1. 저장소 루트의 `index.html`을 브라우저로 직접 연다.
2. 또는 로컬 서버를 띄워 실행한다.

예시:

```bash
python -m http.server 8000
```

브라우저에서 `http://localhost:8000`에 접속하면 된다.

## 사용 방법

1. `한 박스 목표 중량`, `한 박스 개수`, `허용 오차`를 설정한다.
2. 필요하면 포도 무게 범위도 조정한다.
3. `포도 배치` 버튼으로 12개 무게를 생성한다.
4. `조합 시작` 버튼으로 조건에 맞는 박스 조합을 찾는다.

## 결과 해석

- Box 1 ~ Box 4까지는 서로 겹치지 않는 조합만 표시한다.
- 결과 카드에는 총무게, 오차, 개수, 선택된 접시 인덱스가 함께 나온다.
- 조건을 만족하는 조합이 없으면 탐색 실패 메시지가 표시된다.

## 배포 참고

GitHub Pages를 사용할 경우 저장소 설정에 맞춰 정적 파일을 배포하면 된다. 현재 프로젝트는 루트 기준의 단일 페이지 앱 구조이므로, 배포 시 `index.html`, `style.css`, `app.js`가 같은 경로에 있어야 한다.

## 메모

- 무작위 생성은 xorshift32 기반 시드 방식을 사용한다.
- 최적 조합은 오차가 작은 순서로 정렬한 뒤, 서로 겹치지 않는 조합을 우선 선택한다.
